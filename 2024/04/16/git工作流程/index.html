前言

为什么要用 git

  1. 远程仓库：不想电脑进水/着火/死机导致写好的代码没了的话
  2. 版本控制：修改一个 bug、增添一个功能，却发现代码跑不起来，此时想回退已经回退不了
  3. 作为开发最基础的工具之一，现在不用，以后一定用到，那就现在开始吧
  4. 简单，上手快，用多就熟练了

要做哪些准备

  1. 确保你的电脑上安装有 git
  2. 把自己电脑上的公钥发给 github

    网上有很多教程，这里不再重复

流程演示

  1. 找到仓库地址，复制下来

  2. 在一个空白目录下，在git bash或终端git clone刚才的地址
  3. 用 vscode 或者编辑器打开目录，根据 md 文档提示跑起代码，注意要在 demo 文件夹下运行main.py
  4. 上 github 或者看微信有没有分派给自己的 issue 或者未解决的 issue
  5. 根据 issue 描述，新开一个代码的分支


    注意分支的来源，此处是 master 分支

    main 分支存放的是已经确认无误的，有最新功能的，稳定的代码

  6. 在项目文件夹work_together终端下，git fetch获取最新分支信息
  7. 切换到自己创建的分支git checkout xxx

    注意切换前的分支要保存，git push 或者 git stash，自行搜索

  8. 开始写代码…完成后 git 三步走
    1. git add .
    2. git commit -m "提交信息"
    3. git push origin xxx(自己的分支)
  9. 在 github 中发起合并请求，指派给其他人审查合并

    注意要自己的分支合并到 main 分支，别弄错！！

  10. 在群里@Ta 一声，叫他处理合并请求
    如"@L issuexxx已经解决，发起了合并请求，处理一下"
  11. 在其他人合并完后，更新本地 master 分支

    这里我搞混了 main 和 master（

  12. 🎉 你已经完成一个完整流程的开发！接下来可以看其它 issue 哦

git 还有其它很多功能，可以自行学习
